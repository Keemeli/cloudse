# GitHub Repository Checklist

Before submitting, make sure:

## Code
- [ ] All code works and runs without errors
- [ ] .env files are NOT committed (only .env.example)
- [ ] API key is set as environment variable
- [ ] Database connection works

## Documentation  
- [ ] README.md explains what the project does
- [ ] DEPLOYMENT.md has deployment instructions
- [ ] All file paths and URLs are updated for your deployment

## GitHub
- [ ] Code is pushed to GitHub
- [ ] Repository is public (so teacher can access)
- [ ] Repository has a good name (like "weather-cloud-pipeline")
- [ ] README has your name

## Deployment
- [ ] App is deployed to cloud (CSC/AWS/Azure)
- [ ] App URL is accessible from browser
- [ ] Weather data fetching works
- [ ] Database is storing data

## Testing Your App

1. Open your deployed URL
2. Select a city
3. Click "Refresh Data"
4. Check that temperature and weather info appears
5. Look at Statistics section - should show data
6. Check that chart displays temperature over time
